Claude Usage Integration for Home Assistant

Design Spec — April 10, 2026

What This Does

A Home Assistant integration that automatically checks your Claude subscription usage every 5 minutes and shows the data as sensors in HA. You can use these sensors on dashboards, in automations, or anywhere else in Home Assistant.

How You Install It

  1. Add the GitHub repository in HACS (one-time)
  2. Install the integration and restart HA
  3. Go to Settings → Integrations → Add "Claude Usage"
  4. Choose how to connect (see below)

How You Connect It

When you add the integration, you'll see two options:

Option 1 — Login with Claude Preferred

Click a button, get redirected to Claude's login page, sign in, and you're done. Tokens are pulled automatically.

Note: This may not work if Anthropic's servers reject the redirect. If it fails, use Option 2.

Option 2 — Paste a Token Fallback

Copy a refresh token from your PC (from the file ~/.claude/.credentials.json) and paste it into the setup screen. The integration validates it immediately and you're set.

Both options end up in the same place — the integration has what it needs to talk to Claude's servers and will keep the connection alive on its own.

How It Stays Connected

The integration manages its own authentication, completely independent from Claude Code on your PC:

  • Access tokens last about 8 hours
  • Before they expire, the integration automatically gets a new one
  • New tokens are stored securely inside HA
  • If anything goes wrong, it retries on the next poll — no manual intervention needed

What Sensors You Get

All sensors appear under a single device called "Claude Subscription":

⚫
Session Utilization
Your current 5-hour usage window — shown as a percentage (e.g., 44%)
⏳
Session Resets At
When your 5-hour window resets — includes a "minutes until reset" countdown
📈
Weekly Utilization
Your 7-day usage — shown as a percentage (e.g., 13%)
📅
Weekly Resets At
When your 7-day window resets — includes a "minutes until reset" countdown
💰
Extra Credits Used
How many extra-usage credits you've spent this month (e.g., 1,628 of 10,000). Only shows if extra usage is enabled on your plan.
💳
Extra Usage Utilization
Extra credits used as a percentage of your monthly limit (e.g., 16.28%). Only shows if extra usage is enabled on your plan.

Settings You Can Change

After setup, go to the integration's "Configure" button to adjust:

  • Poll interval: How often to check usage (default: every 5 minutes, range: 1–30 minutes)

What Happens When Things Go Wrong

SituationWhat Happens
Token expiresAuto-refreshes before it expires — you never notice
Network issue or API errorKeeps the last known data, retries on the next cycle
Extra usage not on your planThose two sensors simply don't appear
Bad token during setupShows an error message, lets you try again

The integration should never crash or need manual fixes after the initial setup.
